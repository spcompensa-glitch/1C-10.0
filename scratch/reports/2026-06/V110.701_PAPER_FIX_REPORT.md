## 🐛 Problema Identificado

### Discrepâncias no Cálculo de P&L

Após análise dos resultados do sistema, foram identificadas discrepancias significativas no cálculo de P&L (Profit & Loss) para trades ativos no modo PAPER:

**Exemplo de Discrepância:**
- **TRXUSDT SHORT 50x:**
  - Entry: $0.351700 | Target: $0.341149
  - ROI teórico: 15.02% | P&L teórico: $1.50
  - Resultado exibido: **$7.53** (5x maior que o esperado)

- **ETCUSDT SHORT 50x:**
  - Entry: $8.0630 | Target: $7.8211  
  - ROI teórico: 150.0% | P&L teórico: $15.00
  - Resultado exibido: **$4.25** (3.5x menor que o esperado)

### Raiz do Problema

O sistema estava utilizando uma fórmula dinâmica para cálculo de margem:

```python
margin_used = (qty * entry_price) / leverage
```

Isso resultava em margens inconsistentes que não refletiam a realidade do modo PAPER, onde cada slot deve ter uma margem fixa de **$10.00** (10% do capital inicial de $100).

---

## 🛠️ Soluções Implementadas

### 1. Ajuste do Cálculo de Margem no Modo PAPER

**Arquivos Modificados:** `backend/services/okx_rest.py`

#### Mudança 1 - Linha 1867:
```python
# ANTES
margin_used = float(slot.get("entry_margin", 0)) or ((qty_closed * entry_p) / float(slot.get("leverage", 50) or 50))

# DEPOIS  
margin_used = float(slot.get("entry_margin", 0)) or (10.0 if self.execution_mode == "PAPER" else ((qty_closed * entry_p) / float(slot.get("leverage", 50) or 50)))
```

#### Mudança 2 - Linha 1916:
```python
# ANTES
margin = float(s.get("entry_margin") or (qty * entry / 50.0))

# DEPOIS
margin = float(s.get("entry_margin") or (10.0 if self.execution_mode == "PAPER" else (qty * entry / 50.0)))
```

#### Mudança 3 - Linha 2216:
```python
# ANTES
margin = float(p.get("entry_margin") or (qty * entry / 50.0))

# DEPOIS
margin = float(p.get("entry_margin") or (10.0 if self.execution_mode == "PAPER" else (qty * entry / 50.0)))
```

### 2. Lógica de Cálculo Padrão

Todos os cálculos de P&L agora seguem a fórmula padronizada:

```python
p_usd = (roi / 100.0) * margin
```

Onde:
- **ROI:** Calculado pela função `calculate_roi()` existente
- **Margin:** $10.00 fixo para modo PAPER, calculado dinamicamente para modo REAL
- **P&L:** Resultado final em USD
